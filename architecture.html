Acceptance loop — per document loop.py · agents.py

0
Loop budget gate stop if rounds_run ≥ max_total_rounds — hard cap on total challenger rounds across all docs. Else rounds_run += 1.
1
Challenger Opus Reads the source doc → emits a JSON task package: type, skill_tags, context, question (task/workflow), reference, rubric (10–15 weighted criteria). On retry, gets failure feedback and generates a new angle.
2
Quality Verifier — GATE Opus Checks leakage / reasoning-vs-recall / rubric quality / type consistency. FAIL → feedback back to Challenger, next round (before spending solver compute).
3
Solvers ×N samples Haiku (weak) Opus (strong) Both answer the task solver_samples times (variance reduction). Weak is meant to struggle; strong to succeed.
4
Judge Opus, Kimi-style Grades every answer against the rubric only, normalized to [0,1]. Produces weak & strong score lists.
5
Acceptance test Accept only if ALL hold:
  • weak avg ≤ weak_avg_max and weak max ≤ weak_max_max, no zero weak
  • strong avg ≥ strong_min and < strong_max (not trivial)
  • gap (strong − weak) ≥ min_gap
✔ pass → emit accepted example, stop this doc · ✗ fail → label failure mode (TOO EASY / FAILED ON STRONG) → feedback → next round
⏹
Exit Accepted example returned, or max_rounds exhausted for this doc, or global loop budget hit → None.
